Multi-stage builds All languages Production tips
Step 1 — Look at the repo files to find the language
go.mod → Go lang  |  base: golang:VERSION-alpine  |  install: go mod download
package.json → Node/React  |  base: node:VERSION-alpine  |  install: npm install
requirements.txt → Python  |  base: python:VERSION-alpine  |  install: pip install -r requirements.txt
pom.xml → Java Maven  |  base: maven:VERSION-alpine  |  install: mvn dependency:go-offline
Gemfile → Ruby  |  base: ruby:VERSION-alpine  |  install: bundle install
Cargo.toml → Rust  |  base: rust:VERSION-alpine  |  install: cargo build
Step 2 — FROM tells you the package manager
...alpine → use apk add --no-cache   (Alpine Linux — tiny, 5MB)
ubuntu / debian → use apt-get install -y   (Ubuntu/Debian)
amazonlinux → use yum install -y   (Amazon Linux / CentOS)
Step 3 — The 5 questions to answer before writing
1
What language?
Look at files: go.mod / package.json / requirements.txt
2
What version?
Open that file — "go 1.24" or "node: 18" or "python-3.11"
3
How to install dependencies?
go mod download / npm install / pip install -r requirements.txt
4
How to build?
Check README — go build / npm run build / mvn package
5
How to run?
Check README — ./app / node server.js / uvicorn main:app
All Dockerfile instructions explained
InstructionWhat it doesExample
FROM Pick the base image (starting point) FROM golang:1.24-alpine
WORKDIR Set working folder inside container WORKDIR /app
COPY Copy files from your machine into container COPY go.mod go.sum ./
RUN Run a command during BUILD time RUN go mod download
ENV Set environment variable ENV PORT=8080
EXPOSE Document which port the app uses EXPOSE 8080
ARG Variable only available during build ARG APP_VERSION=1.0
USER Switch to non-root user (security) USER appuser
HEALTHCHECK Tell Docker how to check if app is alive HEALTHCHECK CMD curl /health
CMD Default command when container STARTS (can override) CMD ["./app"]
ENTRYPOINT Fixed command — cannot be overridden ENTRYPOINT ["./app"]
Universal template — works for every language
# ── Stage 1: BUILD (big image — compile your code) ── FROM language:version-alpine AS builder # Install system packages if needed (gcc etc.) RUN apk add --no-cache system-package WORKDIR /app # ALWAYS copy dependency file FIRST (Docker layer cache trick) COPY dependency-file . RUN install-command # Then copy rest of source code COPY . . # Build the binary / output RUN build-command # ── Stage 2: RUNTIME (tiny image — just run it) ── FROM minimal-base-image # Create non-root user (security) RUN adduser -D -s /bin/sh appuser WORKDIR /app # Copy ONLY the compiled output from Stage 1 COPY --from=builder /app/binary . USER appuser EXPOSE PORT CMD ["./binary"]
Real examples — fill in the blanks
Go backend
FROM golang:1.24-alpine AS builder RUN apk add --no-cache git WORKDIR /app COPY go.mod go.sum ./ RUN go mod download COPY . . RUN CGO_ENABLED=0 go build \ -ldflags="-w -s" \ -o app ./cmd/main.go FROM alpine:3.19 RUN apk add --no-cache \ ca-certificates tzdata RUN adduser -D appuser WORKDIR /app COPY --from=builder /app/app . USER appuser EXPOSE 8080 CMD ["./app"]
React / Vite frontend
FROM node:18-alpine AS builder WORKDIR /app COPY package*.json ./ RUN npm install COPY . . # Vite → dist/ | CRA → build/ RUN npm run build FROM nginx:alpine RUN rm -rf \ /usr/share/nginx/html/* # Vite output is dist/ COPY --from=builder \ /app/dist \ /usr/share/nginx/html COPY nginx.conf \ /etc/nginx/conf.d/default.conf EXPOSE 80 CMD ["nginx","-g","daemon off;"]
Python FastAPI
FROM python:3.11-alpine AS builder # psycopg2 needs gcc to compile RUN apk add --no-cache \ gcc musl-dev libpq-dev WORKDIR /app COPY requirements.txt . RUN pip install --no-cache-dir \ -r requirements.txt FROM python:3.11-alpine # Python needs interpreter! RUN apk add --no-cache libpq WORKDIR /app COPY --from=builder \ /usr/local/lib/python3.11 \ /usr/local/lib/python3.11 COPY . . EXPOSE 8000 CMD ["uvicorn","main:app", "--host","0.0.0.0", "--port","8000"]
Node.js Express API
# No build step needed! # Node runs source directly FROM node:18-alpine WORKDIR /app COPY package*.json ./ # --only=production skips # devDependencies (smaller!) RUN npm install \ --only=production COPY . . RUN adduser -D appuser USER appuser EXPOSE 3000 # from scripts.start in package.json CMD ["node", "server.js"]
Runtime image — which one to pick
alpine:3.19 5MB — has shell + basic OS. Use for Go, Python, Node binaries. Most common.
scratch 0MB — completely empty. Use for Go binaries with CGO_ENABLED=0. Ultra minimal.
distroless 2MB — no shell, no tools. Most secure. Use in high-security production (your Shivam project!).
nginx:alpine 23MB — Alpine + Nginx. Always use for React/Vue frontend static files.
temurin:JRE Use for Java .jar files. JRE = run only. JDK = compile + run (too big for runtime).

Production tips — do this, not that
never do this
COPY . . RUN npm install # copies everything FIRST # then installs # cache breaks every change!
always do this
COPY package*.json ./ RUN npm install COPY . . # dep file first → install # THEN copy source # cache only breaks when # package.json changes
never do this
RUN apt-get update RUN apt-get install git RUN apt-get install curl # 3 separate layers = bloat # each RUN = new layer
always do this
RUN apk add --no-cache \ git curl wget # one RUN = one layer # --no-cache = no index stored # smaller image
never do this
ENV DB_PASSWORD=mypassword ENV JWT_SECRET=abc123 # hardcoded secrets in image! # anyone can read with # docker inspect
always do this
ENV DB_PASSWORD="" # empty default # pass real value at runtime: docker run -e DB_PASSWORD=xxx # or via K8s secret / .env
never do this
FROM node:18-alpine # running as root by default # if hacker gets in = game over # full container control
always do this
RUN adduser -D appuser USER appuser # non-root user # limited permissions # hacker can't do much
Always create .dockerignore (next to Dockerfile)
# .dockerignore — stops these from being copied into image node_modules/ # huge folder — npm install handles this inside .git/ # git history not needed in container .env # secrets must never go into image! dist/ # build output — will be rebuilt inside *.log # log files README.md # docs not needed Dockerfile # dockerfile itself not needed inside .dockerignore # this file itself
Production checklist — before pushing to ECR/DockerHub
✓
Multi-stage build used
Build image for compiling, tiny image for running
✓
Dependency file copied before source code
go.mod / package.json / requirements.txt first → then COPY . .
✓
Non-root user created
adduser -D appuser → USER appuser before CMD
✓
No secrets hardcoded
Passwords, tokens, API keys passed at runtime via env vars
✓
.dockerignore file exists
node_modules, .env, .git excluded
✓
--no-cache on apk add
Keeps image smaller, no package index stored
✓
EXPOSE port documented
Tells Kubernetes and Docker what port the app uses
✓
CMD uses exec form not shell form
CMD ["./app"] not CMD ./app — handles signals properly in K8s
Quick commands
# Build image docker build -t myapp:v1 . # Run container docker run -p 8080:8080 -e PORT=8080 myapp:v1 # Check image size docker images myapp # See layers and what's taking space docker history myapp:v1 # Open shell inside running container (debug) docker exec -it container-name sh # See logs docker logs -f container-name # Tag for ECR and push docker tag myapp:v1 account.dkr.ecr.region.amazonaws.com/myapp:v1 docker push account.dkr.ecr.region.amazonaws.com/myapp:v1
